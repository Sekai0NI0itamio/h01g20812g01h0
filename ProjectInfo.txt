# YouTube Shorts Automation

This project automates the creation of YouTube Shorts videos using AI-generated scripts, images, background videos, and text-to-speech (TTS) technology.

## Features

- Creates AI-powered YouTube Shorts with dynamic transitions and effects
- Supports both image-based and video-based shorts generation
- Script/query generation through Scitely with DeepSeek models
- Text-to-speech through Groq Orpheus TTS
- Image generation through OpenRouter
- Custom house script template loaded from `automation/prompts/ai_shorts_script_template.txt`
- Background video fetching from Pexels and Pixabay
- Random background music mixing from `Music/`
- Random top-half gameplay / brainrot overlay from `AttentionGrabBrainRotVideos/`
- Stock-image fallback through Unsplash and Pexels when image generation fails
- gTTS fallback for TTS reliability
- Advanced video features:
  - Parallel video processing for faster rendering
  - Word-by-word text animations
  - Eye-level subtitle placement with readable sizing
  - Customizable transitions and effects
  - Smart background video processing
  - Edge blur and background blur options
  - Watermark support
  - Automatic video duration adjustment
  - Hardware acceleration support (NVIDIA GPU)
- Automatic fallback mechanisms for reliability
- Progress logging and error handling

## Provider Note

This codebase now uses Groq for TTS, Scitely for DeepSeek text generation, and OpenRouter for image generation. The default image model is `sourceful/riverflow-v2-pro`.

## Setup

1. **Clone the repository:**

   ```sh
   git clone https://github.com/yourusername/youtube-shorts-automation.git
   cd youtube-shorts-automation
   ```

2. **Install dependencies:**

   Use Python `3.11` or `3.12` for local runs. Python `3.14` is too new for several pinned packages in this project and will trigger source-build failures.

   ```sh
   pip install -r requirements.txt
   ```

   For local `create-only` runs, use the smaller runtime dependency set instead:

   ```sh
   pip install -r requirements-local.txt
   ```

3. **Set up API credentials:**

   - **Local create-only runs with C05:**
     - Start your local C05 provider so `POST http://127.0.0.1:8129/RequestApiKey` is available.
     - Make sure C05 has usable keys configured for `scitely`, `groq`, `openrouter`, `pexels`, `newsapi`, `unsplash`, and `pixabay`.
     - This local mode does not require those provider API keys in `.env`.
     - If you only create videos and do not auto-upload, you also do not need YouTube OAuth or Google Secret Manager credentials.

   - **Google Cloud:**
     - Create a Google Cloud Project.
     - Enable the "Secret Manager API" and "YouTube Data API v3".
     - Create a service account with at least the "Secret Manager Secret Accessor" role.
     - Download the JSON key for this service account. This will be your "master" credential file.
     - Store your other secrets (like your YouTube OAuth `client_secret.json` content and other API keys) in Google Secret Manager within your project. The application expects secrets named `youtube-client-secrets` and `google-credentials`.
   - **OAuth Consent Screen (For YouTube Upload):**
     - In the Google Cloud Console, go to "APIs & Services" -> "OAuth consent screen".
     - Set the User Type to "External".
     - Fill in the required app information (app name, user support email, developer contact).
     - On the "Scopes" page, you don't need to add scopes manually; the app will request them.
     - On the "Test users" page, click "+ ADD USERS" and add the Google account email you will use to authorize the YouTube uploads. **This is a critical step to avoid `access_denied` errors while the app is in "Testing" mode.**
   - Set up a Scitely API key or auth token for DeepSeek content generation if you are not using C05 for local keys
   - Set up a Groq API key for Orpheus TTS if you are not using C05 for local keys
   - Set up an OpenRouter API key for image generation if you are not using C05 for local keys
   - Set up NewsAPI, Pexels, Pixabay, and Unsplash keys if you are not using C05 for local keys

4. **Set up environment variables:**
   Create a `.env` file in the root directory and add the following variables:

   ```env
   # Google Cloud Platform
   GCP_PROJECT_ID=your-gcp-project-id
   # --- IMPORTANT ---
   # Use an ABSOLUTE path to your master service account JSON key file.
   # Example for Linux/macOS: /home/user/keys/my-gcp-key.json
   # Example for Windows: C:\Users\user\keys\my-gcp-key.json
   GOOGLE_APPLICATION_CREDENTIALS=/path/to/your/master-gcp-service-account-key.json

   # API Keys
   # Optional when USE_C05_LOCAL_KEYS=true and C05 is configured with these providers
   SCITELY_API_KEY=your_scitely_key
   SCITELY_MODEL=deepseek-v3.2
   GROQ_API_KEY=your_groq_key
   OPENROUTER_API_KEY=your_openrouter_key
   NEWS_API_KEY=your_news_api_key
   PEXELS_API_KEY=your_pexels_api_key
   PIXABAY_API_KEY=your_pixabay_api_key
   UNSPLASH_API_KEY=your_unsplash_api_key

   # TTS Configuration
   USE_GROQ_TTS=true
   GROQ_TTS_MODEL=canopylabs/orpheus-v1-english
   GROQ_TTS_VOICE=troy
   GROQ_TTS_SPEED=1.15

   # Image Generation
   OPENROUTER_IMAGE_MODEL=sourceful/riverflow-v2-pro
   OPENROUTER_IMAGE_SIZE=1K
   OPENROUTER_SITE_URL=https://your-site.example
   OPENROUTER_APP_NAME=YouTube Shorts Automation

   # Optional local asset directories
   SHORTS_MUSIC_DIR=Music
   SHORTS_BRAINROT_DIR=AttentionGrabBrainRotVideos
   SHORTS_MUSIC_VOLUME=0.08

   # Subtitle / overlay positioning
   SHORTS_SUBTITLE_FONT_SIZE=72
   SHORTS_SUBTITLE_Y_RATIO=0.42
   SHORTS_BRAINROT_HEIGHT_RATIO=0.5

   # Local C05 key provider
   USE_C05_LOCAL_KEYS=true
   C05_BASE_URL=http://127.0.0.1:8129

   # Video Settings
   ENABLE_YOUTUBE_UPLOAD=false
   YOUTUBE_TOPIC=Artificial Intelligence

   # Parallel Processing
   ENABLE_PARALLEL_RENDERING=true
   MAX_PARALLEL_TASKS=4
   ENABLE_GPU_ACCELERATION=true
   ```

5. **Install additional requirements:**
   - FFmpeg (required for video processing)
   - ImageMagick (required for text effects)
   - CUDA toolkit (optional, for GPU acceleration)

## Usage

1. **Basic local usage:**

   ```sh
   python main.py
   ```

   - Interactive local runs will prompt for `create-only` or `auto-upload`.
   - Non-interactive runs default to `create-only`.
   - By default, local runs fetch Scitely, Groq, OpenRouter, NewsAPI, Pexels, Pixabay, and Unsplash keys from C05.

2. **Create a video without uploading:**

   ```sh
   python main.py video --run-mode create-only --topic "Latest robotics breakthrough"
   ```

   Local media directories:
   - Put music tracks in `Music/`
   - Put gameplay / attention-grabber clips in `AttentionGrabBrainRotVideos/`
   - If either folder is empty, the run continues without that layer

   Suggested setup on macOS:

   ```sh
   brew install python@3.11 ffmpeg imagemagick
   /opt/homebrew/bin/python3.11 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements-local.txt
   python main.py video --run-mode create-only --topic "Latest robotics breakthrough"
   ```

3. **Create and auto-upload:**

   ```sh
   python main.py image --run-mode auto-upload --topic "AI chip race"
   ```

   - `auto-upload` still requires valid YouTube OAuth credentials.
   - `create-only` avoids YouTube auth entirely.

4. **Use environment keys instead of C05 for one run:**

   ```sh
   python main.py auto --run-mode create-only --no-use-c05-keys
   ```

5. **Schedule automated runs:**
   ```sh
   python schedule.py
   ```

## Advanced Features

### House Script Template

- The spoken script template lives in `automation/prompts/ai_shorts_script_template.txt`
- The Scitely script request includes that template on each run
- Default house style:
  - no spoken intro
  - direct factual pacing
  - no summary ending
  - exactly one final CTA starting with `Like and subscribe if...`

### Local Media Layers

- Background music is selected randomly from `Music/` and mixed under the final video audio
- Brainrot overlay video is selected randomly from `AttentionGrabBrainRotVideos/`
- The overlay is cropped/resized to the top half of the vertical frame
- Captions are composited after the overlay so they stay on top

### Subtitle Placement

- Subtitle clips render around eye level using `SHORTS_SUBTITLE_Y_RATIO`
- Subtitle size is controlled by `SHORTS_SUBTITLE_FONT_SIZE`
- The default size is intentionally larger than literal 16px because 16px is not readable in a 1080x1920 short

### Parallel Processing System

- **parallel_renderer.py**: Handles parallel video rendering with GPU acceleration
- **parallel_tasks.py**: Manages concurrent task execution and resource allocation
- Features:
  - Multi-process video rendering for faster output
  - Smart clip pre-rendering for complex compositions
  - Hardware acceleration support
  - Memory-efficient processing of large videos
  - Automatic task distribution and load balancing
  - Progress tracking and error recovery

### Image-based Shorts (shorts_maker_I.py)

- Uses OpenRouter image generation with stock-image fallback
- Automatic style selection and prompt enhancement
- Smart fallback to video mode if image generation fails
- Zoom and transition effects for still images

### Video-based Shorts (shorts_maker_V.py)

- Smart video background selection and processing
- Multiple transition effects between scenes
- Word-by-word text animation
- Background blur and edge blur effects

### Helper Utilities

- **image.py**: Advanced image processing and manipulation
- **fetch.py**: Efficient content fetching with caching
- **crossfade.py**: Customizable transition effects
- **text.py**: Text processing and animation utilities
- **audio.py**: Audio processing and enhancement
- **memory.py**: Memory management and optimization
- **process.py**: Process management and monitoring
- **minor_helper.py**: Miscellaneous utility functions
- **blur.py**: Advanced blur effects and filters
